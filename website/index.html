What are Agent Skills
Overview

Portable knowledge for AI agents

Agent Skills are folders containing a SKILL.md file — a blend of metadata and instructions that agents discover and load on demand. Think of them as loadable expertise: domain knowledge, team conventions, and repeatable workflows your agent can access exactly when needed.

The format was originally developed by Anthropic, released as an open standard, and has been adopted by 30+ agent products including Claude, Codex, Cursor, Copilot, Gemini CLI, and more.

SKILL.md — minimal example 
---
name: roll-dice
description: Roll dice using a random number generator. Use when asked
  to roll a die (d6, d20, etc.), roll dice, or generate a random roll.
---

# Skill body — full Markdown, no format restrictions

To roll a die, run this command:

```bash
echo $((RANDOM % <sides> + 1))
```
Why Agent Skills?

Three perspectives

1

For complete newcomers

Find a skill, copy its GitHub URL, paste it in your AI chat. That's it — the AI reads the instructions and knows what to do. No installation, no terminal.

2

For teams and enterprises

Capture organizational knowledge — runbooks, style guides, domain procedures — in portable, version-controlled packages.

3

For skill authors

Build capabilities once and deploy them across any skills-compatible agent. One file, zero infrastructure. Share on GitHub like any project.

How it works — Progressive Disclosure
Discovery

Lightweight scan at startup

At startup, the agent loads only the name and description of every available skill — roughly 50–100 tokens per skill. No instructions are loaded yet.

~project/.agents/skills/
~/.agents/skills/
~/.claude/skills/
Activation

On-demand loading when relevant

When the user's task matches a skill's description, the agent reads the full SKILL.md into context. This is where your instructions, examples, and scripts live.

Recommended: < 5,000 tokens
Recommended: < 500 lines
Execution

Instructions followed, scripts run

The agent follows the Markdown instructions — running referenced scripts, loading additional files from scripts/ or references/ as needed.

scripts/  → executable code
references/ → documentation
assets/   → templates
Quickstart — Create your first skill in 5 minutes
Step 1

Create the directory structure

Skills live in a skills/ folder. The folder name must match the name field in your SKILL.md.

Shell
# Project-level (shared with project)
mkdir -p .agents/skills/my-skill

# User-level (personal, available everywhere)
mkdir -p ~/.agents/skills/my-skill

# Claude Code also reads from:
.claude/skills/my-skill/
~/.claude/skills/my-skill/
Project-level skills override user-level skills with the same name.
Step 2

Write your SKILL.md

Create .agents/skills/my-skill/SKILL.md with YAML frontmatter followed by Markdown instructions.

SKILL.md
---
name: my-skill
description: >
  What this skill does and when to use it.
  Use when the user asks to [specific task].
license: MIT
metadata:
  author: your-name
  version: "1.0"
---

## Instructions

1. First, do this.
2. Then, do that.
Step 3

Write clear instructions

The Markdown body is completely free-form. Structure it so the agent can follow it step-by-step. Recommended sections:

Recommended body structure
## When to use this skill
Trigger conditions, scope, and limitations.

## Instructions
Step-by-step procedure. Be explicit.

## Gotchas
Environment-specific facts. Things that defy expectations.

## Examples
Concrete input → output pairs.

## Validation
How to verify the output is correct.
Step 4

Add scripts (optional)

Place executable files in a scripts/ subdirectory. Agents can run them. Design scripts to be agent-friendly:

Python — PEP 723 inline dependencies
#!/usr/bin/env -S uv run
# /// script
# requires-python = ">= 3.11"
# dependencies = ["requests", "rich"]
# ///
import sys, json

# Design principles for agent scripts:
# - No interactive prompts
# - Structured output (JSON/CSV)
# - Errors to stderr, data to stdout
# - --help flag, meaningful exit codes
print(json.dumps({"result": "ok"}))
SKILL.md Format Reference
Complete Specification

Frontmatter fields

Every SKILL.md starts with YAML frontmatter. Only name and description are required. The Markdown body that follows has no format restrictions — write whatever helps the agent do the task.

name
Required
Lowercase alphanumeric + hyphens only. 1–64 characters. Must match the parent directory name. No consecutive hyphens, no leading/trailing hyphens.
name: pdf-processing
name: code-review
name: data-analysis
description
Required
1–1024 characters. This is the most important field — it's what agents read at startup to decide whether to activate the skill. Describe both what the skill does AND when to use it. Include specific keywords agents would encounter. Use imperative phrasing: "Use this skill when..."
Good: "Extracts text and tables from PDF files. Use when working with PDFs, forms, or document extraction."
Poor: "Helps with PDFs."
license
Optional
License name or reference to a bundled file. Keeps skill packages self-contained.
license: MIT
license: Apache-2.0
license: LICENSE.txt
compatibility
Optional
1–500 characters. Only include if your skill has specific requirements: target agent, system packages, or network access needs. Most skills don't need this field.
Requires git, docker, jq
Designed for Claude Code
Requires Python 3.11+
metadata
Optional
Arbitrary key-value map for additional properties not defined by the spec. Use unique key names to avoid conflicts.
metadata:
  author: your-org
  version: "2.1"
allowed-tools
Optional · Experimental
Space-delimited list of tools pre-approved to run. Support varies between agents. Reduces per-action approval prompts for trusted operations.
allowed-tools: Bash(git:*) Read Write
Complete SKILL.md example 
---
name: pdf-processing
description: >
  Extract text and tables from PDF files, fill PDF forms, and merge
  multiple PDFs. Use when working with PDF documents, forms, or when
  the user mentions PDFs, document extraction, or form filling.
license: Apache-2.0
compatibility: Requires Python 3.11+ and uv
metadata:
  author: example-org
  version: "1.0"
allowed-tools: Bash(uv:*) Read Write
---

## When to use this skill
Activate when the user needs to extract text, fill forms, or merge PDFs.

## Instructions
1. Verify the input is a valid PDF.
2. Run the appropriate script from scripts/.
3. Validate output before returning.

## Gotchas
- Scanned PDFs require OCR (scripts/ocr.py, not scripts/extract.py)
- Password-protected PDFs fail silently without --password
Best Practices
01 — Description

Write descriptions for discovery, not documentation

The description field is the only thing agents read before deciding to activate a skill. Include the exact words users would say. Use imperative triggers like "Use this skill when..."

Test: Write 20 queries — 10 that should activate the skill, 10 that should not. Measure trigger rates and iterate.
02 — Scope

Design coherent, right-sized skills

Too narrow (one-liners) wastes overhead. Too broad (everything about AWS) becomes unfocused. Aim for skills that own a clear domain and activate for a well-defined set of tasks.

Rule of thumb: Under 500 lines, under 5,000 tokens. Split larger skills into referenced files.
03 — Instructions

Add what agents lack; skip what they know

Focus on project conventions, domain-specific procedures, edge cases unique to your environment, and specific tools. Omit general knowledge the agent already has.

Write a Gotchas section for environment-specific facts that defy common assumptions.
04 — Control

Be prescriptive only when it matters

Give freedom when multiple approaches produce equivalent results. Be explicit and strict when operations are fragile, irreversible, or must follow a specific sequence.

Prefer: "Provide defaults, not menus. Procedures over declarations."
05 — Scripts

Make scripts agent-friendly by design

Agents can not respond to interactive prompts. Use non-interactive flags, structured output (JSON/CSV not freeform text), helpful error messages, and meaningful exit codes.

Pattern: Data to stdout, diagnostics to stderr, structured exit codes.
06 — Validation

Build validation loops into instructions

Have the agent verify its own work before returning. Checklists for multi-step workflows, output template matching, and plan-validate-execute patterns for destructive operations.

Example: "Before committing, run tests and confirm green. If tests fail, fix then re-run."
Frequently Asked Questions
What exactly is an Agent Skill?
A skill is a plain-text instruction file (called SKILL.md) that teaches an AI assistant how to do a specific task. Think of it like a recipe card: the AI reads it, follows the steps, and gets the job done. Skills only load when needed, so the AI stays fast.
I'm not a developer — can I still use skills?
Yes, absolutely. Skills are plain-English instruction files, not code. You need an AI assistant (like Claude.ai — free tier works) and nothing else. No terminal, no GitHub account required. Just copy a skill's GitHub URL and paste it into your AI chat.
Are skills free?
Skills themselves are free text files — every skill in this directory is free to download and use. You do need an AI subscription (e.g. Claude Pro, GitHub Copilot) to run them, since skills are read and followed by the AI. Many AI tools have free tiers that work fine.
How do I know if a skill is safe to use?
Every skill in this list is a public GitHub repository — you can read the full instructions before loading anything. Check the repo's stars, last update date, and README. Skills are plain text: there's no executable code that runs on your machine just from loading a skill.
How are Agent Skills different from fine-tuning?
Fine-tuning permanently changes how an AI thinks (expensive and hard to update). Skills are just instruction files you can update, swap, or share at any time without touching the AI itself.
What's the difference between Agent Skills and MCP?
They do different things and work great together: Agent Skills teach the AI how to do something (workflows, best practices). MCP helps the AI access things (APIs, databases, external tools).
Which AI tools support Agent Skills?
Claude (Claude.ai and Claude Code), GitHub Copilot, VS Code, Codex (OpenAI), Antigravity (Google), Gemini CLI, Kiro, and 25+ more. The list keeps growing as more tools adopt the open standard.
How do I create my first skill?
Create a folder named after your skill, put a SKILL.md file inside with a name and description at the top, then write your instructions in plain English below. Place the folder in .claude/skills/ or .github/skills/ depending on your AI tool. Full guide: Creating custom skills.

Discovery & installation

Quick install with CLI tools

Shell — various install methods
# Using the official `skills` CLI
npx skills add anthropics/skills

# Install from any GitHub repo
npx skills add <owner>/<repo>

# Install a specific skill by folder
npx skills add <owner>/<repo> --skill pdf-processing

# Validate a skill locally
npx skills-ref validate ./my-skill

Manual installation

Shell — manual clone
# Clone skill repo into user skills directory
git clone https://github.com/anthropics/skills \
  ~/.agents/skills/anthropics

# Or copy a single skill
cp -r my-downloaded-skill \
  .agents/skills/my-downloaded-skill

# Structure check: skill must have SKILL.md
ls ~/.agents/skills/my-skill/SKILL.md
