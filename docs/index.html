What This Repository Is

Think of **Agent Skills** as a high-fidelity "how-to guide" for AI assistants. Instead of requiring the AI to memorize every workflow upfront, skills provide situational capabilities as they are needed—reducing noise and increasing reliability.

The Problem

Generic agents hallucinate when performing complex, multi-step engineering tasks. They lack the specific domain knowledge of your project's architecture, dependencies, and style guides.

The Solution

Skills are modular instruction sets (SKILL.md) that teach an agent how to execute specific workflows seamlessly across Claude, Antigravity, and other modern agentic environments.

The Standard of Excellence

Not all skills are created equal. We maintain a high bar for inclusion in this repository to ensure agentic stability and peak performance.

Clarity & Precision

Instructions must be unambiguous. Avoid 'try to' or 'maybe'—use direct, imperative language like 'Execute' or 'Generate'.

Operational Scope

A good skill does one thing perfectly. Monolithic skills slow down agents and increase token noise.

Safety & Reliability

Define error handling and edge cases. A skill should know when to stop and ask for human verification.

Proven Examples

Include at least two real-world usage examples in the SKILL.md to ground the agent's behavior.

Preferred Pattern

Explicit Instruction

"When a PR is opened, immediately scan the `packages/core` directory for changes. If changes exist, run `npm test` and output the results as a summary table."

Why? It provides clear triggers, specific targets, and defined output formats.

Anti-Pattern

Vague Suggestion

"Try to look at the code if you have time and maybe let me know if anything looks weird or if there are bugs."

Why? Hallucination-prone. No specific trigger, no defined success criteria.

Using Skills

We've made using skills as simple as adding a dependency to your project. Whether you're using the CLI or manual placement, the process is streamlined for the modern agentic IDE.

Discovery

The agent identifies a gap in its current capabilities and searches for a compatible skill.

Activation

Metadata is parsed from the SKILL.md and cached for immediate situational awareness.

Execution

The agent follows the step-by-step instructions and accesses any local scripts or templates.

CLI Installation

npx skills add anthropics/skills/docx

* Supports GitHub shorthand, full URL, or local relative paths.

Manual Integration

Path: .github/skills/skill-name/

Simply drop your `SKILL.md` (and any scripts) into this directory. Modern agents automatically index these folders on startup.

Creating Skills

Skills are instruction bundles that tell the agent how to perform specific tasks. They are not executable code—they are high-level semantic instructions.

Anatomy of a Skill

A project-ready skill follows this standardized filesystem pattern for maximum agentic discoverability.

skill-name/

A kebab-case folder

├── SKILL.md

Required: The brains

├── scripts/

Optional: Helper logic

└── templates/

Optional: File outputs

The SKILL.md Blueprint

Metadata is parsed by the agent to determine when this skill should be active. Use clear, semantic labels.

---
name: your-skill-name
description: Accurate, clear use cases for the agent.
---

# Instructions

1. Use semantic headers for different stages.

2. Provide explicit step-by-step logic.

3. Define clear exit criteria and failure modes.
